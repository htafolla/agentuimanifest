# AgentUIManifest Governance

**Version:** 1.1  
**Effective Date:** April 11, 2026  

---

## Maintainer

@htafolla is the current maintainer. Responsibilities:

- Review and merge PRs
- Make final decisions on spec changes
- Ensure backward compatibility

When there are 10+ active contributors, governance will transition to a community model.

---

## How to Contribute

1. Open a GitHub issue describing your proposed change
2. Discuss with the community
3. Submit a PR
4. Maintainer reviews and merges

No CLA required. Your PR constitutes agreement to license under Apache 2.0.

---

## Versioning

Semantic versioning: `MAJOR.MINOR.PATCH`

| Change | Process |
|--------|---------|
| Patch (bug fix) | PR, merge by maintainer |
| Minor (new feature) | RFC issue, PR, merge by maintainer |
| Major (breaking) | RFC issue, community discussion, 30-day notice |

---

## Guiding Principles

1. **Open by default** — Anyone can propose changes
2. **Show, don't tell** — Reference implementations preferred
3. **Don't break things** — Backward compatibility matters
4. **Keep it simple** — Don't over-engineer
